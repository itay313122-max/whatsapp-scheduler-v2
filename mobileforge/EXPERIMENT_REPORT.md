# MobileForge — דוח ניסויים: עומס וגיוון

**עודכן:** 2026-06-24 · **סבב:** #1 (בסיס)
**מטרה:** למדוד עד כמה המנוע יכול לייצר אפליקציות מגוונות (צבע, שפה, ז'אנר, פיצ'רים)
באופן יציב ואמין.

> כל הנתונים בדוח הזה מבוססים על רינדור אמיתי (Playwright, headless Chromium)
> שרץ **אופליין לחלוטין** (כל הרשת חסומה) — מוכיח שהאפליקציות self-contained.

---

## מתודולוגיה

לכל אפליקציה: קוד JSX → `buildHtmlDocument()` (טרנספיל בצד-שרת, React מוטמע
מקומית, אפס CDN) → רינדור צילום מסך ב-iPhone (390×844, DPR 3) עם **כל הרשת חסומה**.
נמדדו: זמן build, זמן render, גודל HTML, שגיאות JS בקונסול.

## ממדי גיוון שהוכחו

הניסוי הראה שאפשר לשנות באופן עצמאי:

1. **צבע** — 5 ערכות שונות (כתום, ירוק, סגול, כחול, טורקיז), כולן דרך 4 משתני CSS.
2. **שפה + כיוון** — אנגלית (LTR), עברית (RTL מלא — פריסה מתהפכת), ספרדית (LTR).
   מטבע מתאים אוטומטית: `$` / `₪` / `€`.
3. **ז'אנר** — משלוחי אוכל, כושר, נסיעות (כל אחד עם מבנה מסך שונה לגמרי).
4. **פיצ'רים אינטראקטיביים** — עגלת קניות, טבעת התקדמות SVG, טיימר חי
   (setInterval), שמירת מועדפים, סינון לפי קטגוריה.
5. **תוספות תוכן** — באנר מבצע, חיפוש, כרטיסי תמונה — נוספו בלי לשבור את הקיים.

## ממצאי יציבות

- ✅ **100% רינדור נקי** — אף אפליקציה לא קרסה, אפס שגיאות בקונסול.
- ✅ **אופליין מלא** — כולן רצו עם חסימת רשת מוחלטת (self-contained מאומת).
- ✅ **RTL/LTR** — המעבר בין כיוונים עבד נקי, כולל היפוך פריסת כרטיסים.
- ✅ **Safe areas** — header/nav מכבדים את ה-notch וה-home indicator.
- ⚠️ **לקח שתועד:** `btn-secondary` הוא `width:100%` כברירת מחדל — בתוך כרטיס
  flex צריך `width:auto` כדי לא לדחוס. תוקן ב-App 4.

## הערכת קיבולת (עומס)

- **build + render לאפליקציה:** ~2.4 שנ׳ סך הכל על core בודד.
- מכאן: שרת בודד יכול לייצר ולרנדר **~25 אפליקציות/דקה** ברצף, ויותר במקביל.
- גודל קבוע ~193KB לאפליקציה → זול לאחסון ולהפצה (גם ב-CDN חינמי).
